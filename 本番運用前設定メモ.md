# 本番運用前 設定メモ
## Aoki Design Studio — 見積シミュレーターサイト

> 本番公開前に以下の設定をすべて完了してください。
> 設定ファイルの場所は `js/emailjs-config.js` が中心です。

---

## Phase 1 で必要な設定

### 1. EmailJS（フォーム送信機能）
**ファイル:** `js/emailjs-config.js`

```js
const EMAILJS_CONFIG = {
  PUBLIC_KEY:   "YOUR_PUBLIC_KEY_HERE",   // ← 要変更
  SERVICE_ID:   "YOUR_SERVICE_ID_HERE",   // ← 要変更
  TEMPLATE_ID:  "YOUR_TEMPLATE_ID_HERE",  // ← 要変更
};
```

**取得手順:**
1. https://www.emailjs.com/ でアカウント作成
2. Email Services でGmailなどのサービスを登録 → Service IDをコピー
3. Email Templates でテンプレートを作成 → Template IDをコピー
4. Account > API Keys > Public Key をコピー

**テンプレートで使用できる変数:**
- `{{from_name}}` — 送信者名
- `{{from_email}}` — 送信者メール
- `{{from_phone}}` — 電話番号
- `{{message}}` — 相談内容
- `{{quote_summary}}` — 見積内容（シミュレーター結果）
- `{{reply_to}}` — 返信先メール

### 2. reCAPTCHA v3（スパム対策）
**ファイル①:** `js/emailjs-config.js`

```js
const RECAPTCHA_CONFIG = {
  SITE_KEY: "YOUR_RECAPTCHA_SITE_KEY_HERE", // ← 要変更
};
```

**ファイル②:** `index.html`（2箇所）

```html
<!-- head内 src の render= 部分 -->
<script src="https://www.google.com/recaptcha/api.js?render=YOUR_RECAPTCHA_SITE_KEY_HERE">
```

**取得手順:**
1. https://www.google.com/recaptcha/admin でサイトを登録
2. reCAPTCHA v3 を選択
3. ドメインを登録（例: aoki-design-studio.com）
4. サイトキー（公開）をコピー → 上記2箇所に貼り付け
5. シークレットキー（非公開）はサーバー側検証に使用（現状はフロントのみ）

### 10. 管理ダッシュボードのパスワード変更
**ファイル:** `admin.html`

パスワードはSHA-256ハッシュで保護されています。変更手順：

1. ブラウザのコンソールで以下を実行してハッシュを生成：
```js
crypto.subtle.digest('SHA-256', new TextEncoder().encode('新しいパスワード'))
  .then(h => console.log(Array.from(new Uint8Array(h)).map(b=>b.toString(16).padStart(2,'0')).join('')))
```
2. 出力されたハッシュ文字列を `admin.html` の `ADMIN_PASSWORD_HASH` に設定：
```js
const ADMIN_PASSWORD_HASH = "ここにハッシュ値を貼り付け";
```

また、本番環境では `admin.html` への直接アクセスをサーバー側でIP制限することを推奨します。

## Phase 4 で必要な設定

### 12. Sentry エラーログ設定
**ファイル:** `js/emailjs-config.js`

```js
const SENTRY_CONFIG = {
  DSN:         "YOUR_SENTRY_DSN_HERE", // ← 要変更
  ENVIRONMENT: "production",
  RELEASE:     "1.0.0",
  SAMPLE_RATE: 0.1,
};
```

**取得手順:**
1. https://sentry.io/ でアカウント作成・プロジェクト作成（JavaScript）
2. プロジェクト設定 > Client Keys > DSN をコピー
3. 上記 `DSN` に貼り付け
4. `ENVIRONMENT` は `"production"` のままでOK
5. `RELEASE` はバージョン管理に合わせて変更（例: `"1.0.0"`）
6. `SAMPLE_RATE` はパフォーマンストレースのサンプリング率（0〜1.0）

> DSN が未設定（`YOUR_` を含む）の場合は Sentry に送信せず `console.error` に出力されます。

---

### 13. セキュリティヘッダーの確認（.htaccess / _headers）
デプロイ環境に応じて適切なファイルを使用してください：

| 環境 | ファイル |
|---|---|
| Apache / LiteSpeed サーバー | `.htaccess` |
| Netlify | `_headers` |
| Vercel | `vercel.json` の `headers` セクションに移植 |

本番ドメイン確定後、CSP の `connect-src` / `frame-src` 等に追加ドメインがあれば追記してください。

---

### 14. admin.html のIP制限（推奨）
**ファイル:** `.htaccess`（Apacheの場合）

```apache
# コメントアウトを解除してIPを指定
<Files "admin.html">
  Require ip あなたのIPアドレス/32
</Files>
```

Netlify の場合は Netlify Identity またはパスワード保護機能を利用してください。

---

### 15. A/Bテスト結果の確認方法
GA4 で以下のカスタムイベントを確認：

| イベント名 | 説明 |
|---|---|
| `ab_test_view` | バリアント表示（パラメータ: `test_id`, `variant_id`） |
| `ab_test_conversion` | フォーム送信コンバージョン（パラメータ: `test_id`, `variant_id`, `conversion_type`） |

GA4「探索」レポートでセグメント分析するか、BigQuery エクスポートでCVRを比較してください。

---

### 16. 料金設定の恒久変更
管理ダッシュボード（admin.html）の「料金設定」タブで編集後、「JSONダウンロード」ボタンで出力した `pricing.json` を `data/pricing.json` に上書きすることで、全ユーザーへ永続反映されます。
